Practical 8: Form Validation

Learn to validate user input using HTML5 validation and JavaScript.

1. What is Form Validation?

Concept: Form validation ensures that user input meets specific criteria before it's submitted. This prevents errors and improves data quality.

Types of Validation

  • Client-side: JavaScript validation in the browser (immediate feedback)
  • Server-side: Validation on the server (security)
  • HTML5: Built-in browser validation using attributes

Use Case: Required fields, email format, password strength, date ranges, numeric values.

2. HTML5 Built-in Validation

Concept: HTML5 provides built-in validation attributes that work without JavaScript.

Example: HTML5 Validation Attributes

<input type="email" required>
<input type="number" min="0" max="100">
<input type="text" minlength="3" maxlength="10">
<input type="text" pattern="[A-Za-z]{3}">
Form is valid!

Common HTML5 Attributes:

  • required - Field must be filled
  • type="email" - Must be valid email format
  • min/max - Numeric range
  • minlength/maxlength - Character count
  • pattern - Regular expression pattern

3. JavaScript Validation

Concept: JavaScript provides more control and custom validation logic.

Example: Basic JavaScript Validation

function validateForm() {
    const name = document.getElementById('name').value;
    
    if (name === '') {
        alert('Name is required');
        return false;
    }
    
    return true;
}

5. Password Validation

Concept: Check password strength and confirm password matching.

Example: Password Strength Checker

Password must be at least 8 characters with 1 number and 1 special character
Passwords do not match
Passwords match!

Password Requirements:

  • Minimum 8 characters
  • At least one uppercase letter
  • At least one number
  • At least one special character (!@#$%^&*)

7. Summary

Validation Techniques:

  • HTML5 Validation: required, type, min/max, pattern
  • JavaScript Validation: Custom logic, real-time feedback
  • Visual Feedback: Error/success states, messages
  • Event Handling: submit, input, blur, focus

Common Validation Rules:

Field Type Validation
Name Required, min 2 characters, letters only
Email Required, valid email format
Phone Required, 10 digits
Password Min 8 chars, uppercase, number, special char
Age/Date Valid range, age restrictions

Best Practices:

  • Validate on both client-side (UX) and server-side (security)
  • Provide clear, specific error messages
  • Validate in real-time when possible
  • Show password requirements before user types
  • Use visual indicators (colors, icons) for validation states
  • Don't validate on every keystroke (use debouncing)
  • Always sanitize and validate server-side
