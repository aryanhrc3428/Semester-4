Practical 6: CSS Selectors

Learn different types of CSS selectors to target and style HTML elements.

1. Element Selector

Concept: Selects all elements of a specific type.

Syntax: element { property: value; }

Example: All paragraphs styled

/* Styles ALL p elements */
p {
    color: #333;
    line-height: 1.6;
}

This paragraph is styled by the element selector.

So is this one! Notice the consistent styling.

Use Case: Setting default styles for all headings, paragraphs, or links.

2. Class Selector

Concept: Selects elements with a specific class attribute. Can be used multiple times.

Syntax: .classname { property: value; }

Example: Multiple classes

/* Select elements with class="highlight" */
.highlight {
    background-color: yellow;
    padding: 5px;
}

.important {
    font-weight: bold;
    color: red;
}

This is highlighted text in a sentence.

This entire paragraph is important!

This has both classes applied.

This is a box with the .box class.

Use Case: Reusable styles like buttons, cards, alerts, or highlighted text.

3. ID Selector

Concept: Selects a unique element with a specific ID. Should only be used once per page.

Syntax: #idname { property: value; }

Example: Unique elements

/* Select element with id="header" */
#header {
    background-color: #333;
    color: white;
    padding: 20px;
}
This is the unique header section

This paragraph has special styling with an ID.

Use Case: Unique page sections like #header, #footer, #sidebar, or #main-content.

4. Descendant Selector

Concept: Selects elements that are descendants (children, grandchildren, etc.) of another element.

Syntax: ancestor descendant { property: value; }

Example: Nested elements

/* Select p elements inside .article */
.article p {
    text-indent: 20px;
}

.article .quote {
    font-style: italic;
    border-left: 3px solid #999;
}

This paragraph is inside the article and has text indentation.

This is a quote inside the article with special styling.

This nested paragraph is also affected (descendant).

This paragraph is outside the article - no indentation.

Use Case: Styling elements within specific sections like .article p, .sidebar a, .menu li.

5. Child Selector

Concept: Selects only direct children (not grandchildren) of an element.

Syntax: parent > child { property: value; }

Example: Direct children only

/* Select only direct li children */
.list-direct > li {
    color: blue;
    font-weight: bold;
}
  • Direct child (blue, bold)
  • Direct child (blue, bold)
    • Nested child (normal)
    • Nested child (normal)
  • Direct child (blue, bold)

Use Case: Styling only top-level menu items, first-level list items, or direct children.

6. Adjacent Sibling Selector

Concept: Selects an element that immediately follows another specific element.

Syntax: element1 + element2 { property: value; }

Example: First paragraph after heading

/* Select p that immediately follows h3 */
h3 + p {
    background-color: #e6f3ff;
    border-left: 3px solid #0066cc;
}

Use Case: Styling the first paragraph after a heading, or specific element pairs.

7. General Sibling Selector

Concept: Selects all siblings that follow a specific element (not just immediate).

Syntax: element1 ~ element2 { property: value; }

Example: All paragraphs after h4

/* Select all p siblings that follow h4 */
h4 ~ p {
    color: #666;
    font-size: 14px;
}

Use Case: Styling all paragraphs in a section, or elements following a specific heading.

8. Attribute Selector

Concept: Selects elements based on their attributes or attribute values.

Example: Different attribute selectors

/* Has attribute */
input[type="text"] { border: 2px solid #333; }

/* Starts with */
a[href^="https"] { color: green; }

/* Ends with */
a[href$=".pdf"] { color: red; }

/* Contains */
a[href*="example"] { text-decoration: underline; }

Use Case: Styling external links, file type icons, required fields, or specific input types.

9. Pseudo-class Selectors

Concept: Select elements based on their state or position.

Example: Link states

/* Unvisited link */
a:link { color: blue; }

/* Visited link */
a:visited { color: purple; }

/* Mouse hover */
a:hover { color: red; }

/* Being clicked */
a:active { color: orange; }

Hover over and click this test link to see the states.

Example: Structural pseudo-classes

/* First child */
li:first-child { font-weight: bold; }

/* Last child */
li:last-child { color: red; }

/* Every even row */
tr:nth-child(even) { background-color: #f0f0f0; }
  • First item (bold)
  • Second item
  • Third item
  • Last item (red)
Header 1 Header 2
Row 1 Data
Row 2 Data
Row 3 Data

Use Case: Link states, hover effects, zebra-striped tables, first/last item styling.

10. Pseudo-element Selectors

Concept: Style specific parts of an element or insert content before/after.

Example: First letter and line

/* First letter */
.intro::first-letter {
    font-size: 2em;
    font-weight: bold;
}

/* First line */
.intro::first-line {
    font-weight: bold;
}

This is an introductory paragraph. Notice how the first letter is larger and the first line is bold. This styling is often used in articles and books to create a nice visual effect.

Example: Before and After

/* Add content before */
.quote::before { content: '"'; font-size: 2em; }

/* Add content after */
.quote::after { content: '"'; font-size: 2em; }

This is a famous quote with quotation marks added via CSS.

Use Case: Drop caps, quotation marks, clearfixes, decorative elements, text selection styling.

11. Summary of Selectors

Selector Type Syntax Use Case
Element p Style all elements of a type
Class .classname Reusable styles
ID #idname Unique elements
Descendant ancestor descendant Nested elements
Child parent > child Direct children only
Adjacent Sibling elem1 + elem2 Immediate following sibling
General Sibling elem1 ~ elem2 Any following sibling
Attribute [attr="value"] By attribute values
Pseudo-class :hover, :nth-child() States and positions
Pseudo-element ::before, ::after Parts of elements

Best Practices:

  • Use classes for reusable styles (most common)
  • Use IDs sparingly (only for unique elements)
  • Keep selectors simple and specific
  • Avoid overly complex descendant selectors
  • Use semantic class names
