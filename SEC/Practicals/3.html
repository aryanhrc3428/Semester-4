1. HTML Table Basics

Tables organize data into rows and columns. They're perfect for displaying timetables, schedules, pricing, comparisons, and structured information.

Basic Table Structure

Header 1 Header 2 Header 3
Row 1, Cell 1 Row 1, Cell 2 Row 1, Cell 3
Row 2, Cell 1 Row 2, Cell 2 Row 2, Cell 3

Table HTML Elements:

  • <table> - Container for entire table
  • <tr> - Table row (row of cells)
  • <th> - Table header cell (bold, centered by default)
  • <td> - Table data cell (regular cell)
  • <thead> - Groups header rows
  • <tbody> - Groups body rows
  • <tfoot> - Groups footer rows
<table>
  <tr>
    <th>Header</th>
  </tr>
  <tr>
    <td>Data</td>
  </tr>
</table>

Important Table Attributes:

  • border - Adds border around cells
  • cellspacing - Space between cells
  • cellpadding - Space inside cells
  • colspan - Merge cells horizontally (across columns)
  • rowspan - Merge cells vertically (across rows)
  • width - Table or column width

Example: Using colspan and rowspan

Merged Header (2 columns)
Cell 1 Cell 2
Spans 2 rows Cell A
Cell B
<!-- Merge columns -->
<td colspan="3">Spans 3 columns</td>

<!-- Merge rows -->
<td rowspan="2">Spans 2 rows</td>

5. Best Practices for HTML Tables

Do's ✓

  • Use tables for tabular data (schedules, pricing, comparisons)
  • Include proper <thead>, <tbody>, and <tfoot>
  • Use <th> for headers with proper scope attribute
  • Add descriptive captions or titles
  • Use CSS for styling instead of deprecated attributes
  • Make tables responsive for mobile devices

Don'ts ✗

  • Don't use tables for page layout (use CSS Grid/Flexbox instead)
  • Don't leave table headers empty
  • Don't create overly complex nested tables
  • Don't use deprecated HTML attributes (border, cellpadding, cellspacing)
  • Don't ignore accessibility (screen readers rely on proper table markup)
