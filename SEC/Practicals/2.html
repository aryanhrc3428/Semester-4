Lists, Images, and Link Navigation

Learn how to organize information with lists, display images, and create navigation links on your website.

1. Unordered Lists (<ul>)

Unordered lists use bullet points to display items in no particular sequence. Perfect for features, ingredients, or non-sequential information.

Example: Skills Required for Web Development

  • HTML - Structure and semantic markup
  • CSS - Styling and responsive design
  • JavaScript - Interactivity and functionality
  • Version Control (Git) - Collaborative development
  • Responsive Design - Mobile-first approach

Example: Nested Unordered List (Subcategories)

  • Frontend Technologies
    • HTML5
    • CSS3
    • JavaScript (ES6+)
    • Frameworks (React, Vue, Angular)
  • Backend Technologies
    • Node.js
    • Python
    • Databases (SQL, NoSQL)
  • DevOps Tools
    • Docker
    • Kubernetes
    • CI/CD Pipelines
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

Use case: Feature lists, ingredient lists, bullet-point content, any non-sequential information.

2. Ordered Lists (<ol>)

Ordered lists display items in a numbered sequence. Use when order matters—recipes, steps, rankings, or procedures.

Example: Steps to Create a Website

  1. Plan your website structure and content
  2. Create HTML skeleton with semantic elements
  3. Style with CSS for visual design
  4. Add interactivity with JavaScript
  5. Test across different browsers and devices
  6. Deploy to a web server
  7. Monitor and maintain the website

Example: Recipe (Nested Ordered List)

  1. Preparation Phase
    1. Gather all ingredients
    2. Preheat oven to 350°F
    3. Measure and prepare ingredients
  2. Mixing Phase
    1. Combine dry ingredients in a bowl
    2. Mix wet ingredients separately
    3. Combine wet and dry ingredients
  3. Baking Phase
    1. Pour mixture into baking pan
    2. Bake for 30 minutes
    3. Cool before serving

Example: Ranked List with Different Numbering

Roman Numerals (type="i"):

  1. First Priority - Critical bug fixes
  2. Second Priority - New feature implementation
  3. Third Priority - Code optimization
  4. Fourth Priority - Documentation updates

Alphabetical (type="A"):

  1. Architecture planning
  2. Backend development
  3. Client-side testing
  4. Deployment and launch
<ol>
  <li>First step</li>
  <li>Second step</li>
</ol>

Use case: Step-by-step instructions, rankings, prioritized items, recipes, tutorials.

3. Inserting Images (<img>)

Images enhance your website's visual appeal and communicate information effectively. Use the <img> tag to embed images.

Key Image Attributes:

  • src - The source/path to the image file
  • alt - Alternative text (for accessibility and SEO)
  • width & height - Image dimensions
  • title - Tooltip text when hovering

Example 1: Image with Full Path (Local File)

Local image file: bridge.jpeg
Width: 400px (auto-scale height)
Alt text for accessibility: "Beautiful bridge over a river"

Example 2: Another Local Image

Local image file: router.jpeg
Width: 350px
Alt text for accessibility: "Modern networking router"

Example 3: Image with Fixed Dimensions

300x200px Image Placeholder
Fixed dimensions: 300px × 200px
Maintains exact size regardless of content

Example 4: Responsive Image (Percent-based width)

To make images responsive, use percentage width instead of fixed pixels:

<img src="image.jpg" alt="Description" width="100%" style="max-width: 600px;">

This ensures the image scales with screen size but never exceeds 600px.

<img src="path/to/image.jpg" alt="Image description" width="400" height="300" title="Hover text">

Best Practices:

  • Always include descriptive alt text for accessibility and SEO
  • Optimize image file sizes (use JPEG for photos, PNG for graphics)
  • Use relative paths for local images: ./images/photo.jpg
  • Set width/height to prevent layout shifts

4. Internal Links

Internal links connect different pages or sections within your website. They help with navigation and SEO.

Type 1: Links to Different Pages

Link to another HTML file in the same folder:

<a href="1.html">Go to Practical 1</a>

Click here to view Practical 1 - Text Formatting

Link to a file in a subfolder:

<a href="./images/gallery.html">View Gallery</a>

Link to parent directory:

<a href="../index.html">Back to Home</a>

Type 2: Links to Sections Within Same Page (Anchors)

Use anchor links to jump to different sections on the same page:

<!-- Define a section with an ID -->
<div id="section-name">
  <h2>Section Title</h2>
</div>

<!-- Create a link to that section -->
<a href="#section-name">Jump to Section</a>

Type 3: Link with Title Attribute (Tooltip)

<a href="page.html" title="This appears on hover">Click me</a>

Hover over me to see tooltip

Use case: Navigation between pages, creating table of contents, jump links for long pages.

5. External Links

External links point to resources outside your website. These can be helpful for research, references, or directing users to other services.

Educational Resources:

  • W3Schools HTML Tutorial - Comprehensive HTML reference and tutorials
  • MDN Web Docs - HTML - Official Mozilla documentation
  • Codecademy - Learn HTML - Interactive coding course

Design Inspiration - Abstract Art:

For inspiration and learning about abstract art and design principles:

  • Tate Museum - Artist Collections - Explore modern and abstract art
  • Artsy - Abstract Art Guide - Curated abstract art collections and essays
  • Guggenheim Museum - Contemporary and abstract artwork exhibitions
  • Museum of Modern Art (MoMA) - Extensive collection of modern and abstract works
  • Wikipedia - Abstract Art - Historical overview and information

Important Rules for External Links:

<!-- Use target="_blank" to open in new tab -->
<a href="https://example.com" target="_blank">External Site</a>

Key points:

  • target="_blank" - Opens link in a new tab (doesn't leave your site)
  • Full URL - Must include https:// or http://
  • SEO consideration - Use rel="noopener noreferrer" for security:
    <a href="https://external-site.com" target="_blank" rel="noopener noreferrer">External Link</a>

Example: News & Information

  • Hacker News - Technology and programming news
  • Medium - Technical articles and blogs
  • GitHub - Code repository and developer community

Best Practice: Always test external links to ensure they're valid and relevant. Users expect links to work and be helpful.
