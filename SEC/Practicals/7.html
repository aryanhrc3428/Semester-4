Practical 7: JavaScript Events

Learn to handle user interactions with JavaScript events.

1. What are Events?

Concept: Events are actions that happen in the browser, like clicks, key presses, or mouse movements. JavaScript can "listen" for these events and respond.

Common Event Types

  • Mouse Events: click, dblclick, mouseover, mouseout, mousemove
  • Keyboard Events: keydown, keyup, keypress
  • Form Events: submit, change, focus, blur
  • Window Events: load, resize, scroll

Use Case: Interactive websites, form validation, games, animations, user feedback.

2. Click Event

Concept: The click event fires when a user clicks on an element.

Example: Click Handler

// Add click event listener
element.addEventListener('click', function() {
    // Code to run when clicked
});

Use Case: Buttons, navigation, toggling content, triggering actions.

6. The Event Object

Concept: When an event occurs, JavaScript creates an event object with information about the event.

Example: Event Information

element.addEventListener('click', function(event) {
    console.log(event.target);     // Element clicked
    console.log(event.type);       // Type of event
    console.log(event.clientX);    // X coordinate
    console.log(event.clientY);    // Y coordinate
});

7. Event Delegation

Concept: Attach a single event listener to a parent element to handle events from multiple child elements.

Example: Dynamic List Items

// Parent handles clicks for all children
list.addEventListener('click', function(event) {
    if (event.target.tagName === 'LI') {
        // Handle the click
    }
});

11. Summary

Key Concepts:

  • Events - Actions that occur in the browser
  • Event Listeners - Functions that wait for events
  • Event Object - Contains information about the event
  • Event Delegation - Handle events efficiently

Common Event Types:

Category Events
Mouse click, dblclick, mouseover, mouseout, mousemove
Keyboard keydown, keyup, keypress
Form submit, change, input, focus, blur
Window load, resize, scroll, unload

Best Practices:

  • Use addEventListener instead of inline event handlers
  • Remove event listeners when no longer needed
  • Use event delegation for dynamic content
  • Prevent default behavior when necessary (event.preventDefault())
  • Stop event bubbling when needed (event.stopPropagation())
